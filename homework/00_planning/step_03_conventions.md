# Шаг 3: Conventions

Ориентир: [doc/conventions.md](../../doc/conventions.md)

## Что сделать

Зафиксируйте правила, по которым будете писать код и проверять поведение RAG.

## Какие файлы

- Создать: `doc/conventions.md` (в вашем репозитории-решении)

## Чеклист разделов

- Границы изменений (минимальный diff, KISS)
- Архитектура модулей (ingest/chunk/index/retrieval/ui)
- Правила ответа (только по контексту, отказ без данных)
- Тестовые требования
- Правила по данным и зависимостям

## Как проверить

- Файл существует.
- Правила применимы к каждой будущей итерации.

## Ожидаемый результат

Есть единый стандарт разработки, чтобы не разъезжались стиль и scope.
